# 放置游戏：属性 / 装备 / 道具 / Buff 数据管理设计

## 一、整体思路

- **属性**：由「基础属性 + 装备加成 + 临时 Buff」组成，每种加成都带**来源**，便于做属性明细和战斗/技能计算。
- **装备**：按部位装备，每个部位一件，提供固定属性加成。
- **道具**：背包中的可消耗/可使用的物品，可影响属性（如吃药加攻击）或触发 Buff。
- **Buff**：有时效的临时修正，记录来源（技能/道具/环境等），到期自动移除。

---

## 二、数据结构设计

### 1. 属性 ID 与基础属性

- 用**属性 ID**（如 `atk`、`def`、`hp`、`critRate`）统一表示一种属性。
- 角色有一份**基础属性**（无装备、无 Buff 时的数值），作为所有加成的计算基准。

### 2. 属性修饰类型

- **固定值**（+100 攻击）：直接加减最终数值。
- **百分比**（+10% 攻击）：基于基础或当前值按比例加成，需约定计算顺序（一般：先基础，再固定加成，再百分比）。

建议计算顺序：

```
最终属性 = (基础 + 装备固定值 + Buff 固定值) × (1 + 装备百分比 + Buff 百分比)
```

如需更细，可再拆「基础百分比」「装备百分比」「Buff 百分比」分别乘。

### 3. 装备

- **装备槽位**：如 `weapon`、`armor`、`accessory` 等，每个槽位只能装一件。
- **装备表（静态）**：每条装备有 `id`、名称、槽位、提供的「属性修饰」列表（属性 ID + 数值或百分比）。
- **角色装备（动态）**：当前穿在身上的装备 ID 列表（按槽位存），用于计算「装备带来的属性」和「属性来源」。

### 4. 道具

- **道具表（静态）**：`id`、名称、类型（消耗品/材料等）、使用效果（如：加血、加 Buff、给资源）。
- **背包（动态）**：`{ itemId, count }[]`，或按 `itemId` 为 key 的 Map。
- 使用道具时扣数量，并根据配置施加效果（回血、加 Buff 等），效果再反映到属性上。

### 5. Buff

- **Buff 实例**：`id`（或模板 id）、来源类型（技能/道具/环境）、来源 id、**剩余持续时间**、**属性修饰列表**（属性 ID + 固定值或百分比）。
- 每帧或每秒扣时间，到期从角色 Buff 列表移除，重算属性。

这样：**属性明细 = 基础 + 各装备条目 + 各 Buff 条目**，每条都带来源，方便做 UI 展示和调试。

---

## 三、属性明细与来源展示

- **属性明细**：按属性 ID 汇总，例如「攻击力」= 基础 100 + 武器 50 + 药水 Buff 20。
- **来源**：每条加成绑定 `sourceType`（equipment / buff / passive）和 `sourceId`（装备 id / Buff 实例 id / 技能 id），UI 可显示「来自：xxx 武器」「来自：狂暴药水」。
- **临时 Buff**：单独列表，每条显示名称、剩余时间、影响的属性和数值，到期自动从列表和属性计算中移除。

---

## 四、数据流小结

1. **读表**：装备表、道具表、Buff 模板（名称、描述、属性修饰等）。
2. **角色数据**：基础属性、当前装备（槽位 → 装备 id）、当前 Buff 列表、背包。
3. **计算**：根据「基础 + 装备 + Buff」按公式算最终属性，并生成「每项属性的来源列表」。
4. **表现**：属性面板显示最终值 + 明细；Buff 面板显示当前 Buff 及剩余时间；装备/道具仅做读写和来源展示。

---

## 五、与示例代码的对应关系

- `examples/placement-game-data/` 下的类型定义与 `AttributeManager` 实现了上述模型：
  - 属性 ID、修饰类型、装备槽位、Buff 结构；
  - 最终属性计算与按来源拆解的明细；
  - 可在此基础上扩展装备穿戴、道具使用、Buff 添加/移除和计时。
